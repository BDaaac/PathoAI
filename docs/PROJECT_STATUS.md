# Project Status

## Что уже готово

- Полный анализ биопсии: классификация + сегментация + AI-контекст одним запросом.
- История анализов:
  - просмотр,
  - фильтрация,
  - удаление,
  - экспорт CSV,
  - генерация PDF-отчета.
- Планер:
  - выбор дня в календаре,
  - список задач по выбранному дню,
  - редактирование карточки (название, дата, срочность, комментарий AI).
- RAG-чат:
  - запросы к базе знаний,
  - авто-учет языка (`ru`/`en`),
  - фильтрация низкорелевантного контекста,
  - bypass для приветствий (без лишнего поиска в базе).

## Для чего нужны основные блоки

- Classification (`/api/classify/`): определяет класс ткани и confidence.
- Segmentation (`/api/segment/`): дает маску/overlay, площадь поражения и контуры.
- Full Analysis (`/api/analyze/`): объединяет classification + segmentation + RAG summary.
- RAG Query (`/api/rag/query/`): отвечает на уточняющие медицинские вопросы.
- Report (`/api/report/<id>/`): PDF-отчет по анализу.

## Ограничения

- Если Ollama недоступен, RAG работает в fallback-режиме (менее «умные» ответы).
- Для наилучшего качества AI-ответов нужен доступный LLM provider (например, Ollama).

## Как пользоваться в демо

1. Открыть Full Analysis.
2. Загрузить изображение.
3. Проверить:
   - класс и confidence,
   - сегментацию,
   - AI-интерпретацию.
4. Задать 2-3 уточняющих вопроса в чате.
5. Скачать PDF и показать историю анализов.
